Redis

Redis：Remote Dictionary Server(远程字典服务器)。是完全开源免费的，用C语言编写的，遵守BSD协议，是一个高性能的（key/value）分布式内存数据库，基于内存运行并支持持久化的NoSQL数据库。
特点：

  1. Redis支持数据的持久化，可以将内存中的数据保存在磁盘中，重启的时候可以再次加载进行使用。
  2. Redis不仅仅支持简单的key - value类型的数据，同时还提供list、set、zset、hash等数据结构的存储。
  3. Redis支持数据的备份，即master - slave（主从复制）模式的数据备份。
  4. Redis默认有16个数据库（redis.conf），默认使用的是第0个。
  5. Redis是单线程的！

Redis的基本数据类型

String（字符串）

  • redis中最基本的类型，一个key对应一个value

  • value可以是字符串还可以是数字，可用于：计数器、统计多单位的数量、粉丝数、对象缓存存储

List（列表）

  • 列表是简单的字符串列表，按照插入顺序排序，可以添加一个元素到列表的头部（左边）或者尾部（右边）

  • 可用作消息队列（lpush rpop）、栈（lpush lpop）

Set（集合）

  • 无序集合，set中的值不能重复

Zset (有序集合)

  • 有序的set集合，不同的是每个元素都会关联一个double类型的分数，通过分数对集合中的成员进行

  • 应用场景：1、存储班级成绩、工资表排序。2、普通消息：重要通知、带权重进行判断！3、排行榜应用实现

Hash（集合）

  • 是一个 string 类型的 field 和 value 的映射表，hash 特别适合用于存储对象。

  • key-value（map<key,value>）! value值是一个map集合！

事务

Redis单条命令保证原子性，但是事务不保证原子性！

Redis事务本质：一组命令的集合！一个事务中的所有命令都会被序列化，在事务执行过程中，会按照顺序执行！

一次性、顺序性、排他性！执行一系列的命令！

Redis事务没有隔离级别的概念！

所有的命令在事务中，并没有直接被执行！只有发起执行命令的时候才会执行！

redis事务：

  • 开启事务（multi）
  • 命令入队（）
  • 执行事务（exec）

编译型异常（代码有问题！命令有错！），事务中所有的命令都不会被执行！

运行时异常（1/0），如果事务队列中存在语法性错误，那么执行命令的时候，其他命令是可以正常执行的，错误命令抛出异常！

持久化之rdb和aof

rdb(快照持久化)

触发机制，生成一个dump.rdb文件

手动:

  1. 在客户端执行 bgsave 指令，不会阻塞redis主服务提供服务，redis服务会调用fork来创建一个子进程，然后子进程负责将快照写入磁盘中，而父进程则继续处理命令请求！
  2. 在客户端执行 save 指令，redis服务会在快照创建完毕之前阻塞所有客户端！
  3. 在客户端执行shutdown指令，redis接收到这个指令会自动执行save命令，在save创建快照结束后关闭redis服务！

自动：

  1. 根据redis.conf中配置文件save选项自动触发BGSAVE指令！

如何恢复rdb文件？

只需将rdb文件放在redis启动目录下即可，redis启动时会自动检查dump.rdb并恢复其中的数据！

特点：

  1. 适合大规模的数据恢复！
  2. 对数据完整性要求不高，可以使用！
  3. 需要一定的时间间隔进行操作，如果redis意外宕机了，最近一段时间的数据可能会丢失！
  4. 开启另一个子进程的时候，会占用一定的内存空间！

aof

aof配置默认是关闭状态的

将所有执行过的写操作命令都记录下来，恢复的时候就把这个文件全部再执行一遍！

如果aof文件有错误，redis是无法启动的，需要修复这个配置文件（使用 redis-check-aof –fix xxx.aof 进行修复）

特点：

  1. 每秒同步一次，可能会丢失一秒的数据！（默认配置）
  2. 相对于数据文件，aof远远大于rdb，修复的速度也比rdb慢！
  3. aof运行效率也要比rdb慢，所以redis默认的配置就是rdb持久化！

Redis主从复制

主从复制，是指将一台Redis服务器的数据，复制到其他的Redis服务器。前者称为主节点（master/leader），后者称为从节点（slave/follower），数据的复制是单向的，只能由主节点到从节点。Master以写为主，Slave以读为主。

==默认情况下，每一台Redis服务器都是主节点，一般情况下只需要配置从机，为从机指定一个主机==

注意：

  • 主机可以写，从机不能写只能读！主机中的所有信息和数据，都会自动保存(同步)到从机！
  • 主机断开连接，从机依旧连接到主机的，但是没有写操作，如果主机再次回来，从机依旧可以直接获取到主机写的信息！

哨兵模式

（自动选取主机的模式）

  • 新建并配置哨兵配置文件（==sentinel.conf==）
  • 启动哨兵模式

特点：

  1. 哨兵模式，基于主从复制模式。
  2. 主从可以切换，故障可以转移，系统的可用性更好。
  3. 哨兵模式就是主从模式的升级，手动到自动，更加健壮。
